AI Contact Form Sorter

An n8n workflow that uses Claude to automatically classify, prioritize, and route incoming contact form submissions — before they ever reach your inbox.

n8n Claude AI Supabase Slack
Overview

What This Does

Contact form submissions hit a webhook, get analyzed by Claude for intent, category, and urgency, then get routed accordingly. High-priority items trigger an immediate Slack alert. Everything gets logged to Supabase. The submitter receives an instant acknowledgment. One endpoint, one model call, one clean path — designed to be minimal and practical.

Step by Step

How It Works

Click each step to see implementation details.

1

Incoming Contact Form

Webhook receives POST requests from any contact form. Accepts name, email, message, and source fields.

Endpoint: POST /webhook/contact-intake

The webhook is configured with responseMode: responseNode so the workflow controls what the submitter sees. No authentication required — the endpoint is public by design. Payload is passed downstream as $json.body.
2

Claude Intent Classification

Claude Sonnet analyzes the message and returns structured JSON with intent, category, urgency, summary, and confidence score.

Uses claude-sonnet-4 at temperature: 0.1 for deterministic classification. The structured output parser enforces a JSON schema so downstream nodes always get predictable fields. System prompt defines 6 intent types, 10 categories, and 4 urgency levels. Single model call per submission — no chains, no retrieval, no tools.
3

Priority Router

IF node checks whether urgency is "high" or "critical". Routes to the appropriate downstream path.

Simple boolean gate — urgency == "high" OR urgency == "critical". True path triggers Slack. False path goes straight to storage. No multi-way switch needed at this scope. Easy to extend later with a Switch node for more granular routing.
4

Send High Priority Slack Alert

Posts a formatted message to a designated Slack channel with submission details and classification results.

Message includes: submitter name/email, intent, category, and AI-generated summary. Uses Slack markdown for formatting. Links back to the n8n workflow execution for full context. Only fires for high/critical — no alert fatigue.
5

Save to Supabase

Inserts the original submission plus all classification metadata into a Postgres table via Supabase.

Table: contact_submissions with columns for name, email, message, intent, category, urgency, summary, and confidence. Uses Supabase's auto-generated UUID primary key and created_at timestamp. Data is stored for review, analytics, and potential retraining.
6

Acknowledge Receipt

Returns a JSON response to the form submitter confirming their message was received.

Responds with 200 OK and a confirmation message. Keeps it simple — no classification details are exposed to the submitter. The response is instant from their perspective because the webhook waits for the full pipeline to complete before responding.

Decisions

Why This Architecture

A few deliberate choices that keep this minimal and practical:

Single model call. One Claude invocation with structured output. No chains, no retrieval, no tool use. Classification doesn't need reasoning loops — it needs speed and consistency.

IF node over Switch. At this scope, two paths (alert vs. store) is enough. A Switch node with 5 outputs would over-engineer the routing before there's evidence more granularity helps.

Supabase over stateless. Storing submissions with their classifications creates a feedback loop. You can review accuracy, spot patterns, and eventually fine-tune the prompt with real data.

No auto-replies. The agent acknowledges receipt but never responds to the actual query. Edge cases where it guesses wrong are low-cost when a human always follows up.

Next Steps

Where This Could Go

This is a v1. Some directions worth exploring as the system handles real volume:

Confidence thresholds. If the model returns confidence below 0.7, route to a human review queue instead of auto-classifying.

Multi-channel routing. Replace the IF node with a Switch that routes sales inquiries to a CRM, support requests to a ticketing system, and spam to /dev/null.

Config-driven categories. Move the intent/category taxonomy to a config file or database table so non-technical users can tune it without editing the prompt.

Analytics dashboard. Build a Supabase view that shows submission volume, category distribution, and average confidence over time.
